## `StringAttributeCondition` <a id="StringAttributeCondition"></a>
    

**Appears in:**

- [AttributeCondition](#AttributeCondition)

<p>StringAttributeCondition defines how to evaluate a string attribute.</p>

<table class="table">
<thead><tr><th width="30%">Field</th><th>Type</th><th>Description</th></tr></thead>
<tbody>
    
  
<tr>
<td>
<code>operation</code> <B>[Required]</B>
</td>
<td>
<code>string</code>
</td>
<td>
   <p>Operation determines what comparison or check should be performed.</p>
<p>The valid operations are:</p>
<ul>
<li>&quot;exists&quot;: Checks that the attribute is present (and not an empty string).</li>
<li>&quot;equals&quot;: String equality comparison with ExpectedValue.</li>
<li>&quot;not_equals&quot;: String inequality comparison with ExpectedValue.</li>
<li>&quot;contains&quot;: Checks if the attribute contains ExpectedValue as a substring.</li>
<li>&quot;not_contains&quot;: Checks if the attribute does not contain ExpectedValue.</li>
<li>&quot;regex&quot;: Interprets ExpectedValue as a regular expression (RE2 syntax)
and checks for a match within the attribute.</li>
</ul>
<p>For operations other than &quot;exists&quot;, ExpectedValue must be provided.</p>
</td>
</tr>
<tr>
<td>
<code>expected_value</code> <B>[Required]</B>
</td>
<td>
<code>string</code>
</td>
<td>
   <p>ExpectedValue is required for all operations except &quot;exists&quot;. Its usage
depends on the chosen Operation, e.g. it may represent an exact string
to match, a substring, or a regular expression.</p>
</td>
</tr>
</tbody>
</table>

### Adding Destination to Odigos

There are two primary methods for configuring destinations in Odigos:

##### **Using the UI**

<Steps>
  <Step>
    Use the [Odigos CLI](https://docs.odigos.io/cli/odigos_ui) to access the UI
    ```bash
    odigos ui
    ```
  </Step>
  <Step>
    Click on `Add Destination`, select `Axiom` and follow the on-screen instructions
  </Step>
</Steps>

##### **Using Kubernetes manifests**

<Steps>
  <Step>
    Save the YAML below to a file (e.g. `axiom.yaml`)
    ```yaml
    apiVersion: odigos.io/v1alpha1
    kind: Destination
    metadata:
      name: axiom-example
      namespace: odigos-system
    spec:
      data:
        AXIOM_DATASET: <Dataset>
      destinationName: axiom
      secretRef:
        name: axiom-secret
      signals:
      - TRACES
      - LOGS
      type: axiom

    ---

    apiVersion: v1
    data:
      AXIOM_API_TOKEN: <Base64 Axiom API token>
    kind: Secret
    metadata:
      name: axiom-secret
      namespace: odigos-system
    type: Opaque
    ```
  </Step>
  <Step>
    Apply the YAML using `kubectl`
    ```bash
    kubectl apply -f axiom.yaml
    ```
  </Step>
</Steps>

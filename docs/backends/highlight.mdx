### Adding Destination to Odigos

There are two primary methods for configuring destinations in Odigos:

##### **Using the UI**

<Steps>
  <Step>
    Use the [Odigos CLI](https://docs.odigos.io/cli/odigos_ui) to access the UI
    ```bash
    odigos ui
    ```
  </Step>
  <Step>
    Click on `Add Destination`, select `Highlight` and follow the on-screen instructions
  </Step>
</Steps>

##### **Using Kubernetes manifests**

<Steps>
  <Step>
    Save the YAML below to a file (e.g. `highlight.yaml`)
    ```yaml
    apiVersion: odigos.io/v1alpha1
    kind: Destination
    metadata:
      name: highlight-example
      namespace: odigos-system
    spec:
      data: {}
      destinationName: highlight
      secretRef:
        name: highlight-secret
      signals:
      - TRACES
      - LOGS
      type: highlight

    ---

    apiVersion: v1
    data:
      HIGHLIGHT_PROJECT_ID: <Base64 Highlight Project ID>
    kind: Secret
    metadata:
      name: highlight-secret
      namespace: odigos-system
    type: Opaque
    ```
  </Step>
  <Step>
    Apply the YAML using `kubectl`
    ```bash
    kubectl apply -f highlight.yaml
    ```
  </Step>
</Steps>

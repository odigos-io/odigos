## Obtaining Lightstep Access Token

Go to **⚙️ > Access Tokens** and click **Create New**

## Configuring Lightstep Backend

Add a new backend with the access token you just created.


## Deploying using yaml

### Applying the Configuration
Save the below YAML to a file (e.g., `destination.yaml`) and apply it using kubectl:

```bash
kubectl apply -f destination.yaml
```
Optional fields are commented out; be sure to uncomment any relevant fields before applying.


```yaml
apiVersion: odigos.io/v1alpha1
kind: Destination
metadata:
  name: lightstep-example
  namespace: <odigos namespace>
spec:
  data: {}
  destinationName: lightstep
  secretRef:
    name: lightstep-secret
  signals:
  - TRACES
  type: lightstep

---
apiVersion: v1
data:
  LIGHTSTEP_ACCESS_TOKEN: <base64 Access Token>
kind: Secret
metadata:
  name: lightstep-secret
  namespace: <odigos namespace>
type: Opaque
```

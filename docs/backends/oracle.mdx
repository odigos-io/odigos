- **ORACLE_ENDPOINT** `string` : OTLP HTTP Endpoint.
  - This field is required
- **ORACLE_DATA_KEY** `string` : Data Key.
  - This field is required

### Adding Destination to Odigos

There are two primary methods for configuring destinations in Odigos:

##### **Using the UI**

<Steps>
  <Step>
    Use the [Odigos CLI](https://docs.odigos.io/cli/odigos_ui) to access the UI
    ```bash
    odigos ui
    ```
  </Step>
  <Step>
    Click on `Add Destination`, select `Oracle APM` and follow the on-screen instructions
  </Step>
</Steps>

##### **Using Kubernetes manifests**

<Steps>
  <Step>
    Save the YAML below to a file (e.g. `oracle.yaml`)
    ```yaml
    apiVersion: odigos.io/v1alpha1
    kind: Destination
    metadata:
      name: oracle-example
      namespace: odigos-system
    spec:
      data:
        ORACLE_ENDPOINT: <OTLP HTTP Endpoint>
      destinationName: oracle
      secretRef:
        name: oracle-secret
      signals:
      - TRACES
      - METRICS
      type: oracle

    ---

    apiVersion: v1
    data:
      ORACLE_DATA_KEY: <Base64 Data Key>
    kind: Secret
    metadata:
      name: oracle-secret
      namespace: odigos-system
    type: Opaque
    ```
  </Step>
  <Step>
    Apply the YAML using `kubectl`
    ```bash
    kubectl apply -f oracle.yaml
    ```
  </Step>
</Steps>

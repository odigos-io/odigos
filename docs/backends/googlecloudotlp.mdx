### Configuring Destination Fields

<Accordion title="Supported Signals:">
  ‚úÖ Traces
  ‚ùå Metrics
  ‚ùå Logs
</Accordion>

- **GCP_PROJECT_ID** `string` : GCP Project ID. The GCP project to send OTLP data.
  - This field is required
  - Example: `myproject`
- **GCP_BILLING_PROJECT** `string` : GCP Billing Project ID. The GCP project to count toward billing and quota.
  - This field is optional
  - Example: `myproject`
- **GCP_APPLICATION_CREDENTIALS** `string` : Google Application Credentials. The JSON GCP Application Credentials file for the GCP project. Either a Workload Identity Federation config file or Service Account Key. Required if running Odigos outside of Google Cloud.
  - This field is optional
  - Example: `Credentials config....`

### Adding Destination to Odigos

There are two primary methods for configuring destinations in Odigos:

##### **Using the UI**

<Steps>
  <Step>
    Use the [Odigos CLI](https://docs.odigos.io/cli/odigos_ui) to access the UI
    ```bash
    odigos ui
    ```
  </Step>
  <Step>
    Click on `Add Destination`, select `Google Cloud Observability (OTLP)` and follow the on-screen instructions
  </Step>
</Steps>

##### **Using Kubernetes manifests**

<Steps>
  <Step>
    Save the YAML below to a file (e.g. `googlecloudotlp.yaml`)
    ```yaml
    apiVersion: odigos.io/v1alpha1
    kind: Destination
    metadata:
      name: googlecloudotlp-example
      namespace: odigos-system
    spec:
      data:
        GCP_PROJECT_ID: <GCP Project ID>
        # Note: The commented fields below are optional.
        # GCP_BILLING_PROJECT: <GCP Billing Project ID>
        # GCP_APPLICATION_CREDENTIALS: <Google Application Credentials>
      destinationName: googlecloudotlp
      signals:
      - TRACES
      type: googlecloudotlp
    ```
  </Step>
  <Step>
    Apply the YAML using `kubectl`
    ```bash
    kubectl apply -f googlecloudotlp.yaml
    ```
  </Step>
</Steps>

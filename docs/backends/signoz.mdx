## Configuring SigNoz Backend

In order to add a new SigNoz backend, insert the SigNoz URL into Odigos UI.


## Deploying using yaml

### Applying the Configuration
Save the below YAML to a file (e.g., `destination.yaml`) and apply it using kubectl:

```bash
kubectl apply -f destination.yaml
```
Optional fields are commented out; be sure to uncomment any relevant fields before applying.


```yaml
apiVersion: odigos.io/v1alpha1
kind: Destination
metadata:
  name: signoz-example
  namespace: <odigos namespace>
spec:
  data:
    SIGNOZ_URL: <Collector Endpoint>
  destinationName: signoz
  signals:
  - TRACES
  - METRICS
  - LOGS
  type: signoz
```
